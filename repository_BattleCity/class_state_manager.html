Detailed Description

Предоставляем методы контроля, регистрации и смены режимов программы

Выступает в роли контроллера правил последовательной смены состояний программы и спользуется при их: создании, удалении, активации и деактивации. В конкретной м случае класс StateManager является реализацией компонента "Контекст" паттерна "Состояние". Имеет интерфейс, через который "Клиент" (Game) обращается к режимам.

Note
Данная реализация допускает наличие параллельных состояний, набор которых изменяем.
Для обозначения этого набора дальше в описании используется термин "список исполняемых состояний".
общий интерфейс состояний BaseState предоставляет возможность управления свойствами
прозрачности и параллельности ( StateBase::SetTranscendent и StateBase::SetTransparent). StateManagers осуществляет обновления с учетом этих свойств.
