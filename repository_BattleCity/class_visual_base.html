Detailed Description

"Component" паттерна "Компоновщик" для визуальных элементов

При наличии необходимости раполагать графические элементы программы, согласно иерархии принадлежности обычно рекомендуется использовать паттерн "компоновщик". класс VisualBase предоставляет интерфейс, общения с абстрактными графическими элементами.
